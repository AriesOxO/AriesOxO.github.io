# 喵呜博客整体设计方案

## 1. 定位与愿景

### 1.1 博客定位

**个人对外分享平台**，覆盖技术开发、开源项目、生活、摄影、思考等多个方向。面向三类受众：

| 受众 | 来访动机 | 核心需求 |
|------|---------|---------|
| 技术同行 | 搜索技术文章、了解开源项目 | 高质量技术内容、项目文档 |
| 朋友/熟人 | 了解近况、看生活分享 | 说说动态、摄影作品 |
| 潜在合作者/雇主 | 评估技术能力和个人风格 | 关于页、开源项目、技术文章 |

### 1.2 品牌调性

- **名称**：喵呜（meow）
- **风格**：技术专业 + 二次元趣味 + 生活温度
- **视觉**：forest 主题（薄荷绿基调），Live2D 看板娘，几何装饰元素
- **语言**：简体中文为主

---

## 2. 信息架构

### 2.1 导航结构

采用**全平铺一级导航**，7 个入口，不使用下拉二级菜单。

```
博客  归档  开源  摄影  说说  关于  友链
```

理由：
- 桌面端导航栏空间充裕（当前仅使用约 40%）
- 移动端为汉堡菜单展开列表，7 项无压力
- 个人博客应让访客一眼看到全部内容类型
- 每个项都对应独立的展示形态，不可合并

### 2.2 各页面定位

| 导航项 | URL | 定位 | 展示形态 |
|--------|-----|------|---------|
| **博客** | `/` | 核心内容入口，文章时间流 | 分页文章列表 + 顶部 Hero |
| **归档** | `/archive.html` | 按年份/标签检索全部文章 | 时间线 + 标签筛选 |
| **开源** | `/projects.html` | 开源项目展示 | 卡片列表 → Landing Page |
| **摄影** | `/photography.html` | 摄影作品画廊 | 瀑布流/网格 + Lightbox |
| **说说** | `/memos.html` | 碎片想法、日常记录 | 时间线 |
| **关于** | `/about.html` | 个人名片 | 结构化自我介绍 |
| **友链** | `/friendLink.html` | 博客互链 | 卡片网格 |

### 2.3 内容分类策略

| 内容类型 | 载体 | 理由 |
|---------|------|------|
| 技术文章 | `_posts/` + 技术标签 | 博客核心，标签区分（Java、Rust、Docker 等） |
| 开源项目 | `_projects/` | 独立 Landing Page + 三栏文档布局 |
| 摄影作品 | `_data/photos.yml` + 独立页面 | 图片内容需要瀑布流布局，与文字内容形态完全不同 |
| 生活随笔 | `_posts/` + 「生活」标签 | 本质是长文，与技术文章展示形态相同，用标签区分 |
| 深度思考 | `_posts/` + 「思考」标签 | 同上，归档页标签筛选可充当独立入口 |
| 碎片想法 | `_data/memos.yml` | 短内容，说说时间线 |
| 读书笔记 | `_posts/` + 「读书」标签 | 长文形态，归入博客 |

**核心原则**：展示形态不同才建独立页面，主题不同用标签区分。

### 3.2 归档页（优化）

**现状**：127 篇文章全部展开，页面极长。
**目标**：按年份折叠，提升浏览效率。

**改造要点**：
- 默认只展开最新年份，其余年份折叠显示年份标题 + 文章数量
- 点击年份标题展开/收起
- 标签云筛选功能保留

```

  ...

► 2025  (45 篇)
► 2024  (50 篇)
► 2023  (20 篇)
```

### 3.3 开源项目页（已完成）

参见 [开源项目 Landing Page 设计方案](open-source-landing-page.md)。

**当前状态**：
- 列表页：卡片网格，二次元文案
- 详情页：Hero + Features + 三栏文档布局（左导航 + 正文 + 右目录）

## 4. 全局 UI/交互优化

### 4.1 Live2D 看板娘适配

| 场景 | 当前行为 | 优化方案 |
|------|---------|---------|
| 桌面端常规页面 | 右下角 150×300 | 保持不变 |
| 移动端 | 同尺寸，遮挡内容 | 缩小到 80×160 或隐藏 |
| 项目文档页 | 同尺寸 | 隐藏（文档页需要右下角空间） |

**实现**：在 `footer.html` 的 Live2D 初始化脚本中根据页面 layout 和屏幕宽度判断。

### 4.2 背景几何装饰

当前所有页面都有三角形、圆形、波浪线等装饰元素。

| 页面类型 | 建议 |
|---------|------|
| 首页、说说、友链、关于 | 保留，增添趣味 |
| 归档（内容密集） | 降低透明度 50% |
| 项目文档页 | 不显示（已通过独立 layout 实现） |

### 4.3 Footer 优化

当前 footer 布局（左 1/4 计数器 + 右 3/4 居中信息）已在本次优化中完成。

后续可考虑在 footer 加入**快捷链接区**（RSS、站点地图等），但优先级低。

## 7. 设计原则

贯穿所有页面的设计决策准则：

1. **展示形态不同才建独立页面，主题不同用标签区分** — 避免页面膨胀
2. **内容为王，装饰为辅** — 几何装饰、看板娘不应干扰阅读
3. **渐进增强** — 先占位后填充，摄影页可以先有骨架再添内容
4. **一致性** — 所有页面共享 header/footer、forest 主题色、字体层级
5. **移动优先验证** — 每个改动都需检查移动端表现
6. **维护成本最小化** — 优先数据驱动（yml/md），避免硬编码 HTML
